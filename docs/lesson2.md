# Rust 引用与生命周期基础

> 基于 `borrow_lifecycle` 项目的 3 个场景，理解引用的本质、借用规则和悬垂引用问题。

## 核心概念

| 概念 | 说明 |
|------|------|
| 引用 `&T` | 指向数据的指针，不拥有数据 |
| 借用 | 通过引用"借"数据来用，不转移所有权 |
| 悬垂引用 | 引用指向已被释放的内存，Rust 编译器禁止这种情况 |

---

## 场景 1：引用是什么

**文件：** `src/reference_basics.rs`

```rust
let s1 = String::from("hello");
let r1 = &s1;  // r1 借用 s1，不拥有数据

println!("s1: {}", s1);   // ✅ s1 还是主人
println!("r1: {}", r1);   // ✅ r1 只是看看
```

**要点：** 引用就像"借书"——`s1` 是书的主人，`r1` 只是借来翻翻。`r1` 离开作用域时，数据不会被销毁，因为它不拥有数据。

**内存示意：**

```text
栈                     堆
┌─────┐          ┌───────────┐
│ s1 ─┼─────────→│ "hello"   │  s1 拥有数据
└─────┘          └───────────┘
┌─────┐               ↑
│ r1 ─┼───────────────┘        r1 只是指向同一块数据
└─────┘
```

## 场景 2：借用的两条规则

**文件：** `src/borrow_rules.rs`

```rust
let mut s = String::from("hello");

// ✅ 规则1：多个人可以同时「看」
let r1 = &s;
let r2 = &s;
println!("两个人同时看: {}, {}", r1, r2);

// ✅ 规则2：只能一个人「改」
let r3 = &mut s;
r3.push_str(", world");
println!("一个人改完了: {}", r3);
```

**要点：** 借用规则总结为两条：

| 规则 | 描述 | 类比 |
|------|------|------|
| 多个 `&T` 可共存 | 同时持有多个不可变引用 | 多人同时看同一本书 |
| `&mut T` 独占 | 可变引用同一时刻只能有一个 | 只有一个人能拿笔改书 |

上面代码之所以合法，是因为 `r1` 和 `r2` 在 `println!` 之后就不再使用（NLL），所以后续可以安全地创建 `r3` 可变引用。

## 场景 3：悬垂引用（不能返回对局部数据的引用）

**文件：** `src/no_dangling.rs`

### 正确写法：返回拥有的数据

```rust
fn create_greeting(name: &str) -> String {
    let greeting = format!("你好, {}!", name);
    greeting  // 把数据交出去（所有权转移），安全
}

let msg = create_greeting("小明");
println!("{}", msg);  // 输出: 你好, 小明!
```

### 错误写法：返回局部变量的引用

```rust
// ❌ 编译不通过！
fn bad_greeting(name: &str) -> &str {
    let greeting = format!("你好, {}!", name);
    &greeting  // greeting 马上就没了，引用指向空气！
}
```

**要点：** 函数内创建的数据在函数结束时会被销毁。如果返回它的引用，就会产生**悬垂引用**（dangling reference）——指向已被回收的内存。Rust 编译器会直接拒绝编译。

正确做法：返回数据本身（所有权转移），而不是返回引用。

```text
函数结束时发生了什么？

create_greeting:  greeting → 所有权转移给调用者 → ✅ 数据活着
bad_greeting:     greeting → 被销毁 → &greeting 指向空气 → ❌ 悬垂引用
```

## 速查表

```text
&T          不可变引用    只读    可以同时存在多个
&mut T      可变引用      可写    同一时刻只能有一个
返回 String  所有权转移    安全    数据跟着返回值走
返回 &T     引用         危险    必须确保数据活得够久（生命周期）
```

## 一句��总结

> 引用是"借来看看"的指针，Rust 编译器通过借用规则和悬垂检查，确保每个引用永远指向有效的数据。
